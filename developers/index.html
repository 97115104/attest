▸ 1. Add attribution to your content

In any content's metadata, add an attribution field:

attribution: collab claude opus 4
ValueMeaning
humanWritten entirely by a human
collab <model>Human + AI collaboration
ai <model>Fully AI-generated
Attribution values

human - Written entirely by a human

collab <model> - Human + AI collaboration

ai <model> - Fully AI-generated

▸ 2. Generate attestations

curl

curl "https://attest.97115104.com/api/create?content_name=My+Doc&model=claude-opus-4&role=collaborated"

node.js

const res = await fetch(
  "https://attest.97115104.com/api/create?" +
  new URLSearchParams({
    content_name: "My Doc",
    model: "claude-opus-4",
    role: "collaborated",
    author: "your-name"
  })
);
const { attestation, urls } = await res.json();
console.log(urls.verify); // shareable verification link

python

import requests

r = requests.get("https://attest.97115104.com/api/create", params={
    "content_name": "My Doc",
    "model": "claude-opus-4",
    "role": "collaborated",
    "author": "your-name"
})
data = r.json()
print(data["urls"]["verify"])  # shareable verification link

All three return the same JSON. Open access, free forever.

▸ 3. Quick start with an agent

Copy and paste the following prompt into any AI agent (Claude, ChatGPT, Gemini, Copilot, or any other). The agent will walk you through creating an attestation by asking what you need it for and verifying the details before generating it.

I need you to help me create a content attestation using the
attest protocol. Walk me through it step by step:

1. Ask me what content I want to attest (title, filename, or
   description).
2. Ask me whether the content was written entirely by a human,
   co-authored with AI, or fully AI-generated.
3. If AI was involved, ask me which model was used.
4. Ask me for an author name (or use "Anonymous").
5. Confirm the details with me before proceeding.
6. Once I confirm, make this HTTP request and show me the result:

   GET https://attest.97115104.com/api/create
     ?content_name={title from step 1}
     &role={authored | collaborated | generated}
     &model={model from step 3, or "Human" if fully human}
     &author={author from step 4}

7. From the JSON response, give me:
   - the verify URL so I can share it
   - the attestation JSON so I can store it alongside my content
   - a suggestion for where to put it (meta tag, frontmatter,
     or a .json file next to the content)

The agent handles the entire flow. Just paste, answer the questions, and you get a working attestation.

       $ curl /api/create
       {
         "ok": true,
         "verify": "..."
       }
       $ _

▸ Create Attestation

GET /api/create

roleNoauthored | collaborated | generated (default: assisted)
authorship_typeNohuman | collab | ai (auto-derived from role)

Parameters

author - Author name (default: Anonymous)

▸ Shorten URL

POST /api/shorten

Store attestation data in the database and get a short link.

curl -X POST https://attest.97115104.com/api/shorten \
  -H "Content-Type: application/json" \
  -d '{"data":"eyJ2ZXJzaW9uIjoiMi4wIi..."}'

Response

{"shortUrl":"https://attest.97115104.com/s/abc12345","shortId":"abc12345"}

Short URLs redirect to the verify page with the attestation data. Open access, always free.

▸ Verify

Attestations are self-verifying. The data URL contains the full attestation as base64 JSON:

https://attest.97115104.com/verify/?data=eyJ2ZXJzaW9uIjoiMi4wIi...

Short URLs (/s/{id}) redirect to the verify page automatically.

       +----------+
       | GET /api |-------> { json }
       +----------+
            |
       +----------+
       | POST /s  |-------> /s/abc123
       +----------+

▸ Attestation v2.0 Schema

{
  "version": "2.0",
  "id": "2026-04-02-a1b2c3",
  "content_name": "My Blog Post",
  "timestamp": "2026-04-02T12:00:00.000Z",
  "platform": "attest.97115104.com",
  "authorship_type": "collab",        // human | collab | ai
  "model": "claude-opus-4",           // or "Human"
  "role": "collaborated",             // authored | collaborated | generated
  "content_hash": "sha256:abcdef...", // SHA-256 of full content
  "document_type": "markdown",
  "author": "97 115 104",
  "signature": {
    "type": "hmac-sha256",
    "value": "4b2d5967...",
    "algorithm": "HMAC-SHA256",
    "data_to_sign": "{\"content_hash\":...}"
  },
  "signer": {
    "name": "97 115 104",
    "id": "97115104"
  }
}

▸ Content hash

SHA-256 of the full file content including frontmatter, prefixed with sha256:.

sha256:$(shasum -a 256 post.md | cut -d' ' -f1)

▸ Signature

HMAC-SHA256 over a JSON string of key attestation fields, using a shared signing key.

data_to_sign = JSON.stringify({
  content_hash, model, timestamp,
  authorship_type, role
})

signature = HMAC-SHA256(signing_key, data_to_sign)
        {
          "v": "2.0",
          "type": "collab",
          "sig": "4b2d...",
          "hash": "sha256:..."
        }

▸ GitHub Actions

Auto-generate attestations on every push to _posts/:

name: Generate Attestations
on:
  push:
    paths: ['_posts/**']
  workflow_dispatch:

jobs:
  attest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: node _scripts/generate-attestations.js
      - run: |
          git config user.name "attest-bot"
          git config user.email "bot@attest.97115104.com"
          git add _data/attestations.json
          git diff --cached --quiet || git commit -m "Update attestations"
          git push

▸ find attestations (scraping/crawling)

attestations can live in several places. here's where to look:

HTML meta tags
<!-- in <head> -->
<meta name="attestation" content="base64...">
<meta name="attribution" content="collab claude opus 4">
<link rel="attestation" href="/attestations/post.json">
JSON files
# common locations:
/attestations/*.json
/_data/attestations.json
/.well-known/attestations.json

# each is a standard attest v2.0 JSON object
frontmatter
---
title: my post
attribution: collab claude opus 4
attestation: https://attest.97115104.com/s/abc12345
---
verify URLs
# full URL (data param is base64 JSON):
https://attest.97115104.com/verify/?data=eyJ2ZXJz...

# short URL (redirects to verify):
https://attest.97115104.com/s/{shortId}

# to decode: atob(data) → JSON.parse()

▸ pseudo prompt

paste this into any agent's system prompt, .instructions.md, tool config, or rules file:

## attestation protocol

when you create or modify content, generate an attestation:

  GET https://attest.97115104.com/api/create
    ?content_name={filename or title}
    &model={your model name}
    &role={authored|collaborated|generated}
    &author={your agent name}
    &content={optional: raw content for hash}

append the returned verify URL to your output or
commit message. open access, free forever, one HTTP call.

example (generated file):
  curl "https://attest.97115104.com/api/create?\
    content_name=output.md&model=gpt-4&\
    role=generated&author=my-agent"

example (human-AI collab):
  curl "https://attest.97115104.com/api/create?\
    content_name=readme.md&model=claude-opus-4&\
    role=collaborated"

store the attestation JSON alongside your output
or link the verify URL in comments/metadata.
