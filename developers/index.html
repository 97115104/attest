▸ 1. Add attribution to your content

In any content's metadata, add an attribution field:

attribution: collab claude opus 4
ValueMeaning
humanWritten entirely by a human
collab <model>Human + AI collaboration
ai <model>Fully AI-generated
Attribution values

human - Written entirely by a human

collab <model> - Human + AI collaboration

ai <model> - Fully AI-generated

▸ 2. Generate attestations

Use the API or a generation script to process your content:

curl "https://attest.97115104.com/api/create?content_name=My+Doc&model=claude-opus-4&role=collaborated"

Returns JSON immediately. No auth needed. Free forever.

▸ Create Attestation

GET /api/create
ParamRequiredDescription

modelNoAI model used (default: gpt-4)

Parameters

model - AI model used (default: gpt-4)

role - authored | collaborated | generated

authorship_type - human | collab | ai (auto-derived)

curl "https://attest.97115104.com/api/create?content_name=README&model=claude-opus-4&role=collaborated"

▸ Shorten URL

POST /api/shorten

Store attestation data in the database and get a short link.

curl -X POST https://attest.97115104.com/api/shorten \
  -H "Content-Type: application/json" \
  -d '{"data":"eyJ2ZXJzaW9uIjoiMi4wIi..."}'

Response

{"shortUrl":"https://attest.97115104.com/s/abc12345","shortId":"abc12345"}

Short URLs redirect to the verify page with the attestation data. Free. No auth.

▸ Verify

Attestations are self-verifying. The data URL contains the full attestation as base64 JSON:

https://attest.97115104.com/verify/?data=eyJ2ZXJzaW9uIjoiMi4wIi...

Short URLs (/s/{id}) redirect to the verify page automatically.

▸ Attestation v2.0 Schema

{
  "version": "2.0",
  "id": "2026-04-02-a1b2c3",
  "content_name": "My Blog Post",
  "timestamp": "2026-04-02T12:00:00.000Z",
  "platform": "attest.97115104.com",
  "authorship_type": "collab",        // human | collab | ai
  "model": "claude-opus-4",           // or "Human"
  "role": "collaborated",             // authored | collaborated | generated
  "content_hash": "sha256:abcdef...", // SHA-256 of full content
  "document_type": "markdown",
  "author": "97 115 104",
  "signature": {
    "type": "hmac-sha256",
    "value": "4b2d5967...",
    "algorithm": "HMAC-SHA256",
    "data_to_sign": "{\"content_hash\":...}"
  },
  "signer": {
    "name": "97 115 104",
    "id": "97115104"
  }
}

▸ Content hash

SHA-256 of the full file content including frontmatter, prefixed with sha256:.

sha256:$(shasum -a 256 post.md | cut -d' ' -f1)

▸ Signature

HMAC-SHA256 over a JSON string of key attestation fields, using a shared signing key.

data_to_sign = JSON.stringify({
  content_hash, model, timestamp,
  authorship_type, role
})

signature = HMAC-SHA256(signing_key, data_to_sign)

▸ GitHub Actions

Auto-generate attestations on every push to _posts/:

name: Generate Attestations
on:
  push:
    paths: ['_posts/**']
  workflow_dispatch:

jobs:
  attest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: node _scripts/generate-attestations.js
      - run: |
          git config user.name "attest-bot"
          git config user.email "bot@attest.97115104.com"
          git add _data/attestations.json
          git diff --cached --quiet || git commit -m "Update attestations"
          git push
